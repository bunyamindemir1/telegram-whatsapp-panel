# Internationalization (i18n)

## Coverage

| Tier | Locales | Notes |
|------|---------|-------|
| **Complete** | `en`, `tr` | Master (`en.json`) and full Turkish review |
| **Community** | `ar`, `de`, `es`, `fr`, `it`, `ja`, `ko`, `nl`, `pl`, `pt`, `ru`, `uk`, `zh` | Key parity with EN; some strings may match English until translated |

Current key count: **533** (see `locales/en.json`). CI validates that every locale file contains the same keys (`scripts/validate_locales.py`). Turkish strings must differ from English for UI keys (`tests/test_i18n.py`).

Helper scripts in `scripts/` (`gen_*.py`, `finish_locales.py`) bootstrap community locales from English. **New strings are always added to `en.json` and `tr.json` first**, then synced to other files.

## User-facing language selection

1. Sidebar language dropdown (persisted in `localStorage` + cookie)
2. URL: `?lang=tr`
3. Browser `Accept-Language` on first visit

## Developer workflow

### Add a string

1. Add key to `locales/en.json`
2. Add Turkish translation to `locales/tr.json`
3. Sync other locales:

```bash
cd src
python3 - <<'PY'
import json
from pathlib import Path
loc = Path("locales")
en = json.loads((loc / "en.json").read_text(encoding="utf-8"))
for p in loc.glob("*.json"):
    if p.name == "en.json": continue
    data = json.loads(p.read_text(encoding="utf-8"))
    for k, v in en.items():
        data.setdefault(k, v)
    p.write_text(json.dumps(data, ensure_ascii=False, indent=2) + "\n", encoding="utf-8")
PY
python scripts/validate_locales.py
pytest tests/test_i18n.py -q
```

4. Use in HTML: `data-i18n="my.key"` or in JS: `tt('my.key')`

### API errors

Backend returns stable keys from `app/error_codes.py` (e.g. `error.schedule.futureRequired`). Add matching entries to `en.json` and `tr.json` under the `error.*` namespace.

### API

```http
GET /api/i18n/locales
GET /api/i18n/tr
```
