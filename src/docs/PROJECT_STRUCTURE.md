# Project Structure

```
telegram-whatsapp-panel/          ← GitHub root (5 visible items)
├── README.md
├── Makefile                      thin wrapper → src/Makefile
├── .gitignore
├── .github/                      CI, templates, governance
└── src/                          all project code
    ├── LICENSE
    ├── Makefile
    ├── app/                      FastAPI backend
    ├── config/                   requirements.txt, pytest.ini
    ├── docker/                   Dockerfile + compose.yml
    ├── docs/                     guides, CONTRIBUTING, SECURITY
    ├── locales/                  15 JSON translation files
    ├── scripts/                  setup, install, dev tools
    ├── static/ templates/
    ├── tests/                    pytest + Playwright E2E
    └── whatsapp-bridge/          Node.js Baileys service
```

## Why so few root files?

GitHub requires `.github/` and `.gitignore` at the repository root. Everything else — license, Docker Compose, backend, frontend, tests — lives under `src/` so the repo landing page stays clean.

## Runtime data (not in git)

| Path | Purpose |
|------|---------|
| `.env` | Secrets (generated by `make setup`) |
| `data/` | SQLite database |
| `sessions/` | Telegram session files |
| `.venv/` | Local Python environment |

## Key commands

All commands run from the repo root:

```bash
make setup    # Docker install + start
make quick    # Local install + start (no Docker)
make test     # unit tests
make e2e      # browser tests
```
